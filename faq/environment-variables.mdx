要为应用程序配置环境，您可能需要设置环境变量。本指南提供了在 macOS (Shell) 和 Windows (PowerShell 及 Windows 命令提示符) 中设置环境变量的说明。

## macOS

### 在 Shell 中设置环境变量

#### 临时环境变量

这些环境变量仅在当前 Shell 会话中有效。

```shell
export VARIABLE_NAME="value"
```

显示环境变量：

```shell
echo $VARIABLE_NAME
```

#### 永久环境变量

要使环境变量在会话间持久化，请将其添加到 Shell 配置文件（例如 `.bashrc`、`.bash_profile`、`.zshrc`）。

对于 Zsh：

```shell
echo 'export VARIABLE_NAME="value"' >> ~/.zshrc
source ~/.zshrc
```

显示环境变量：

```shell
echo $VARIABLE_NAME
```

## Windows

### 在 PowerShell 中设置环境变量

#### 临时环境变量

这些环境变量仅在当前 PowerShell 会话中有效。

```powershell
$env:VARIABLE_NAME = "value"
```

显示环境变量：

```powershell
echo $env:VARIABLE_NAME
```

#### 永久环境变量

要使环境变量在会话间持久化，请将其添加到 PowerShell 配置文件脚本（例如 `Microsoft.PowerShell_profile.ps1`）。

```powershell
notepad $PROFILE
```

在配置脚本中添加以下行：

```powershell
$env:VARIABLE_NAME = "value"
```

保存并关闭文件，然后重新加载配置文件：

```powershell
. $PROFILE
```

显示环境变量：

```powershell
echo $env:VARIABLE_NAME
```

### 在 Windows 命令提示符中设置环境变量

#### 临时环境变量

这些环境变量仅在当前命令提示符会话中有效。

```cmd
set VARIABLE_NAME=value
```

显示环境变量：

```cmd
echo %VARIABLE_NAME%
```

#### 永久环境变量

要使环境变量在会话间持久化，可以使用 `setx` 命令：

```cmd
setx VARIABLE_NAME "value"
```

注意：使用 `setx` 设置环境变量后，需要重新启动命令提示符或任何需要读取新环境变量的应用程序。

在新的命令提示符会话中显示环境变量：

```cmd
echo %VARIABLE_NAME%
```

通过以上步骤，您可以在 macOS Shell、Windows 命令提示符和 PowerShell 中有效设置和显示环境变量，确保为应用程序正确配置环境。
