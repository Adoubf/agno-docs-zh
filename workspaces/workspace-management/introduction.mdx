title: 简介
---
**Agno Workspaces** 是标准化代码库，用于通过Docker在本地运行智能体系统，并在AWS上生产环境部署。它们帮助我们将智能体系统作为代码进行管理。

![工作区](/images/workspace.png)

## 创建新工作区

运行 `ag ws create` 创建新工作区，该命令会要求选择起始模板和工作区名称。

<CodeGroup>

```bash 创建工作区
ag ws create
```

```bash 创建智能体应用
ag ws create -t agent-app-aws -n agent-app
```

```bash 创建 Agent API
ag ws create -t agent-api-aws -n agent-api
```

</CodeGroup>

## 启动工作区资源

运行 `ag ws up` 启动（即创建）工作区资源

<CodeGroup>

```bash 终端
ag ws up
```

```bash 速记
ag ws up dev:docker
```

```bash 完整选项
ag ws up --env dev --infra docker
```

```bash 短选项
ag ws up -e dev -i docker
```

</CodeGroup>

## 设置现有工作区

## 命令选项

<Note>运行 `ag ws up --help` 查看所有选项</Note>

### 环境 (`--env`)

使用 `--env` 或 `-e` 标识过滤环境（dev/prd）

<CodeGroup>

```bash 标识
ag ws up --env dev
```

```bash 速记
ag ws up dev
```

```bash 简短选项
ag ws up -e dev
```

</CodeGroup>

### 基础设施 (`--infra`)

使用 `--infra` 或 `-i` 标识过滤基础设施（docker/aws/k8s）

<CodeGroup>

```bash 标识
ag ws up --infra docker
```

```bash 速记
ag ws up :docker
```

```bash 简短选项
ag ws up -i docker
```

</CodeGroup>

### 资源组 (`--group`)

使用 `--group` 或 `-g` 标识按资源组过滤

<CodeGroup>

```bash 标识
ag ws up --group app
```

```bash 全部选项
ag ws up \
  --env dev \
  --infra docker \
  --group app
```

```bash 速记
ag ws up dev:docker:app
```

```bash 简短选项
ag ws up \
  -e dev \
  -i docker \
  -g app
```

</CodeGroup>

### 名称 (`--name`)

使用 `--name` 或 `-n` 标识按资源名称过滤

<CodeGroup>

```bash 标识
ag ws up --name app
```

```bash 完整选项
ag ws up \
  --env dev \
  --infra docker \
  --name app
```

```bash 速记
ag ws up dev:docker::app
```

```bash 短选项
ag ws up \
  -e dev \
  -i docker \
  -n app
```

</CodeGroup>

### 类型 (`--type`)

使用 `--type` 或 `-t` 标识按资源类型过滤

<CodeGroup>

```bash 标识
ag ws up --type container
```

```bash 完整选项
ag ws up \
  --env dev \
  --infra docker \
  --type container
```

```bash 速记
ag ws up dev:docker:app::container
```

```bash 短选项
ag ws up \
  -e dev \
  -i docker \
  -t container
```

</CodeGroup>

### 试运行 (`--dry-run`)

使用 `--dry-run` 或 `-dr` 标识进行**试运行**。`ag ws up -dr` 仅打印资源信息，不会实际创建。

<CodeGroup>

```bash 标识
ag ws up --dry-run
```

```bash 完整选项
ag ws up \
  --env dev \
  --infra docker \
  --dry-run
```

```bash 速记
ag ws up dev:docker -dr
```

```bash 短选项
ag ws up \
  -e dev \
  -i docker \
  -dr
```

</CodeGroup>

使用 `--debug` 或 `-d` 标识显示调试日志

<CodeGroup>

```bash 标识
ag ws up -d
```

```bash 完整选项
ag ws up \
  --env dev \

```

```bash 速记

```


</CodeGroup>

使用 `--force` 或 `-f` 标识强制重建镜像和容器

<CodeGroup>

```bash 标识
ag ws up -f
```

```bash 完整选项
ag ws up \
  --env dev \
  --infra docker \
  -f
```

```bash 速记

```


</CodeGroup>
