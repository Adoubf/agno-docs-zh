Phidata supports using Singlestore as a storage backend for Agents using the `S2AgentStorage` class.

## Usage

```python storage.py
from phi.storage.agent.singlestore import S2AgentStorage

# SingleStore Configuration
USERNAME = getenv("SINGLESTORE_USERNAME")
PASSWORD = getenv("SINGLESTORE_PASSWORD")
HOST = getenv("SINGLESTORE_HOST")
PORT = getenv("SINGLESTORE_PORT")
DATABASE = getenv("SINGLESTORE_DATABASE")
SSL_CERT = getenv("SINGLESTORE_SSL_CERT", None)

# SingleStore DB URL
db_url = f"mysql+pymysql://{USERNAME}:{PASSWORD}@{HOST}:{PORT}/{DATABASE}?charset=utf8mb4"
if SSL_CERT:
    db_url += f"&ssl_ca={SSL_CERT}&ssl_verify_cert=true"

# Create a storage backend using the Singlestore database
storage = S2AgentStorage(
    # store runs in the ai.agent_runs table
    table_name="agent_runs",
    # db_engine: Singlestore database engine
    db_engine=db_engine,
    # schema: Singlestore schema
    schema=DATABASE),
)

# Add storage to the Agent
agent = Agent(storage=storage)
```

## Params

<ResponseField name="table_name" type="str">
  Name of the table to be used.
</ResponseField>
<ResponseField name="schema" type="Optional[str]" default="ai">
  Schema name, default is "ai".
</ResponseField>
<ResponseField name="db_url" type="Optional[str]" default="None">
  Database URL, if provided.
</ResponseField>
<ResponseField name="db_engine" type="Optional[Engine]" default="None">
  Database engine to be used.
</ResponseField>
<ResponseField name="schema_version" type="int" default="1">
  Version of the schema, default is 1.
</ResponseField>
<ResponseField name="auto_upgrade_schema" type="bool" default="False">
  If true, automatically upgrades the schema when necessary.
</ResponseField>
